What the agent can do
How to read this page

Every tool the agent can call is listed below, grouped by category. Each entry shows:

  • Python function — the concrete call the executor dispatches to.
  • Source — whether inputs come from a SPEC data file, the autonomy DB, a SPEC log, the live SPEC session, the filesystem, or the output of another tool.
  • Inputs / Outputs — the schema the LLM sees, plus a one-line note on what the tool returns.
  • Depends on — tools that typically need to be called first.

Tools that issue a command to the SPEC session are marked with a SPEC badge and summarized in the first section. Every SPEC-mutating call is written to the action log before dispatch.
