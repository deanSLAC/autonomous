BEAMLINE 15-2 · SPEAR3
Overview

Autonomous beamline agent.

A system that takes an experiment configuration, operates the beamline end-to-end, evaluates data as it collects, and reports progress to the user over Slack.

SSRL · BL 15-2 01
Scope

What the agent does.

Given an experiment configuration, the agent configures the beamline, aligns the spectrometer, aligns the user's sample holder, and runs the scan queue. It evaluates each scan as it collects, makes beamline corrections to maintain data quality, and keeps the user informed over Slack. At the end of the run it posts a summary report.

Operates unattended
Runs overnight without a scientist on the floor.
Keeps the user informed
Slack updates on signal-to-noise and progress; morning report summarizing overnight collections.

End-to-end flow

From experiment config to morning report.

01 · INPUT
Experiment config
Samples, edge, quality targets.
02 · SETUP
Configure beamline
Mono, slits, filters, detector.
03 · ALIGN
Spectrometer & sample
Peak I₀, center each sample.
04 · COLLECT
Run scan queue
Scans per sample, top-off aware.
05 · EVALUATE
Inspect & correct
Check SNR, drift; re-tune as needed.
06 · REPORT
Slack updates
Live notifications; morning summary.

The agent iterates between collect and evaluate until the run's quality targets are met or the allocated beam time is exhausted.

Capabilities

Capabilities, in detail.

STEP 01
Configure the beamline
Reads the experiment config, sets mono energy, slits, filters, detector mode.
STEP 02
Align the spectrometer
Peaks up I₀, tunes the monochromator, resolves Bragg offsets.
STEP 03
Align the sample holder
Finds each sample on the wheel, centers it in the beam, stores positions.
STEP 04
Collect data
Queues scans, manages detector saturation, handles top-off gaps.
STEP 05
Reason about results
Inspects each scan live. Judges SNR, edge shape, reproducibility.
STEP 06
Change course
More averages on noisy samples. Re-align when drift detected. Re-prioritize.

The agent reads the config and composes a run plan:
Time budget
18 scans × ~10 min, plus alignment and scheduled top-offs. ≈ 3h 40m.
Drift-aware ordering
Sample wheel re-alignment scheduled based on drift detected in the data.
Quality target
SNR > 250 at the edge. Additional averages added until target is met.
How it starts05
Per-scan evaluation

How the agent judges each scan.

S-A2 · Fe K-edge · scan 3 of 6
SNR = 284 · E₀ drift +0.04 eV
After every scan the agent checks three things and acts on the result.
Signal quality
Computes SNR, edge-step ratio, post-edge flatness. If below target, adds averages.
Beamline stability
Checks E₀ drift against the reference foil. If above 0.2 eV, re-tunes the mono.
Sample position
Watches I₀ and fluorescence stability. If drifting, re-centers the holder.

Every edit is attributed
Plan history shows who changed what, when, and why — Slack user, web session, or agent auto-recovery.

Operational constraints

Limits on agent behavior.

Hardware limits enforced externally
All motion passes through the existing EPICS soft-limit system. The agent cannot override it.
Beam-time budget is hard
The agent stops at the allocated end time and escalates before overshooting.
Every action logged
All corrections, re-alignments, and parameter changes are recorded in the run log with a justification.
Floor operator has priority
Operator commands from the control room take precedence. The agent yields and resumes afterwards.
